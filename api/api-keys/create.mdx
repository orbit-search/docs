---
title: "Create a developer API key for the Orbit Search API"
sidebarTitle: "Create Key"
description: "Generate a personal or organization-scoped Orbit Search API key with specific scopes, an optional expiry date, and optional team restrictions."
---

API keys let your applications call the Orbit Search API without a user session. You create keys using your own user access token, then distribute the resulting `sk_orb_...` key to the service that needs it. Each key carries a set of scopes that limits what it can do — so a read-only integration gets a read-only key.

<Warning>
  The raw key (`payload.key`) is returned **only once** at creation time. Copy it into a secret manager immediately. All subsequent API responses show only `display_key` (e.g. `sk_orb_a0186...55fd`), which cannot be used to authenticate.
</Warning>

## Endpoints

<Tabs>
  <Tab title="Personal key">
    ```
    POST /v2/api-keys
    ```
  </Tab>
  <Tab title="Organization key">
    ```
    POST /v2/organizations/:organizationId/api-keys
    ```
  </Tab>
</Tabs>

Both endpoints require a normal user session Bearer token. Developer API keys cannot create, list, revoke, or delete other keys.

## Request parameters

### Path parameters

<ParamField path="organizationId" type="string">
  The UUID of the organization. Required only when creating an organization key.
</ParamField>

### Body parameters

<ParamField body="name" type="string" required>
  A human-readable label for the key, for example `"production search"`. Shown in list responses and the `display_key` metadata to help you identify keys without exposing the secret.
</ParamField>

<ParamField body="scopes" type="string[]" required>
  List of permission scopes to grant. Supported values:

  - `"search:read"` — call the non-streaming smart search endpoint.
  - `"profile:read"` — fetch a public developer profile DTO by ID.
</ParamField>

<ParamField body="expires_at" type="string">
  ISO 8601 datetime at which the key expires, for example `"2026-12-31T23:59:59.000Z"`. If omitted, the key does not expire unless the operator has configured a default TTL. Requested expirations beyond the operator-configured maximum TTL are rejected.
</ParamField>

<ParamField body="environment" type="string">
  An optional environment label (e.g. `"production"` or `"staging"`). Organization keys only.
</ParamField>

<ParamField body="team_ids" type="string[]">
  Restrict the key to one or more teams within the organization. When set, the key can only search corpora granted to those teams. Organization keys only.
</ParamField>

## Response fields

<ResponseField name="status" type="string" required>
  Always `"success"` on a successful response.
</ResponseField>

<ResponseField name="payload" type="object" required>
  <Expandable title="payload properties" defaultOpen={true}>
    <ResponseField name="id" type="string" required>
      UUID that uniquely identifies this key. Use it in revoke and delete requests.
    </ResponseField>

    <ResponseField name="key" type="string" required>
      The full raw API key (e.g. `sk_orb_a0186REDACTED55fd`). **Returned once only.** Store it in a secret manager before discarding this response.
    </ResponseField>

    <ResponseField name="prefix" type="string" required>
      The non-secret prefix portion of the key (e.g. `sk_orb_a0186`).
    </ResponseField>

    <ResponseField name="suffix" type="string" required>
      The non-secret suffix portion of the key (e.g. `55fd`).
    </ResponseField>

    <ResponseField name="display_key" type="string" required>
      A safe display representation combining prefix and suffix (e.g. `sk_orb_a0186...55fd`). Use this in UIs and support workflows.
    </ResponseField>

    <ResponseField name="name" type="string" required>
      The label you provided at creation time.
    </ResponseField>

    <ResponseField name="scopes" type="string[]" required>
      The scopes granted to this key.
    </ResponseField>

    <ResponseField name="created_at" type="string" required>
      ISO 8601 timestamp of when the key was created.
    </ResponseField>

    <ResponseField name="expires_at" type="string">
      ISO 8601 expiry timestamp. `null` if the key was created without an expiry and no operator default TTL applies.
    </ResponseField>
  </Expandable>
</ResponseField>

## Examples

<Tabs>
  <Tab title="Personal key">
    <CodeGroup>

    ```bash curl
    curl -X POST "$API_BASE/v2/api-keys" \
      -H "Authorization: Bearer $USER_ACCESS_TOKEN" \
      -H "Content-Type: application/json" \
      -d '{
        "name": "production search",
        "scopes": ["search:read", "profile:read"],
        "expires_at": "2026-12-31T23:59:59.000Z"
      }'
    ```

    ```javascript JavaScript
    const response = await fetch(`${API_BASE}/v2/api-keys`, {
      method: "POST",
      headers: {
        Authorization: `Bearer ${USER_ACCESS_TOKEN}`,
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        name: "production search",
        scopes: ["search:read", "profile:read"],
        expires_at: "2026-12-31T23:59:59.000Z",
      }),
    });

    const { payload } = await response.json();
    // Store payload.key immediately — it will not be shown again
    console.log(payload.key);
    ```

    </CodeGroup>
  </Tab>

### Success response

```json
{
  "status": "success",
  "payload": {
    "id": "uuid",
    "key": "sk_orb_a0186REDACTED55fd",
    "prefix": "sk_orb_a0186",
    "suffix": "55fd",
    "display_key": "sk_orb_a0186...55fd",
    "name": "production search",
    "scopes": ["search:read", "profile:read"],
    "created_at": "2026-04-29T00:00:00.000Z",
    "expires_at": "2026-12-31T23:59:59.000Z"
  }
}
```

## Error codes

| Code | HTTP status | Description |
| ---- | ----------- | ----------- |
| `invalid_api_key_request` | 400 | Unsupported scope, name too long, invalid `expires_at`, or expiry beyond the configured maximum TTL. |
| `api_key_management_requires_user_auth` | 403 | The request was made with a developer API key instead of a user access token. Key management always requires user authentication. |
