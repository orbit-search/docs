---
title: "GET /v2/api-keys — list your Orbit Search API keys"
sidebarTitle: "List Keys"
description: "Retrieve metadata for all your active and inactive Orbit Search API keys, including scopes, last-used timestamps, and expiry status."
---

The list endpoint returns metadata for every key associated with your account. You can use it to audit what keys exist, check which are still active, and identify keys that are approaching expiry or have not been used recently. Raw key values are never included — you will only see the `display_key` (prefix and suffix) that you can safely share in UIs and support workflows.

## Endpoint

```
GET /v2/api-keys
```

This endpoint requires a normal user session Bearer token. Developer API keys cannot call key management endpoints.

## Response fields

<ResponseField name="status" type="string" required>
  Always `"success"` on a successful response.
</ResponseField>

<ResponseField name="payload" type="object" required>
  <Expandable title="payload properties" defaultOpen={true}>
    <ResponseField name="keys" type="object[]" required>
      Array of API key metadata objects. Each object contains the following fields:

      <Expandable title="key object properties" defaultOpen={true}>
        <ResponseField name="id" type="string" required>
          UUID that uniquely identifies this key. Pass it to the revoke or delete endpoints.
        </ResponseField>

        <ResponseField name="prefix" type="string" required>
          The non-secret prefix portion of the key (e.g. `sk_orb_a0186`).
        </ResponseField>

        <ResponseField name="suffix" type="string" required>
          The non-secret suffix portion of the key (e.g. `55fd`).
        </ResponseField>

        <ResponseField name="display_key" type="string" required>
          A safe display representation combining prefix and suffix (e.g. `sk_orb_a0186...55fd`). Use this in UIs and support workflows to identify a key without revealing the secret.
        </ResponseField>

        <ResponseField name="name" type="string" required>
          The human-readable label assigned at creation time.
        </ResponseField>

        <ResponseField name="scopes" type="string[]" required>
          The permission scopes granted to this key.
        </ResponseField>

        <ResponseField name="created_at" type="string" required>
          ISO 8601 timestamp of when the key was created.
        </ResponseField>

        <ResponseField name="last_used_at" type="string">
          ISO 8601 timestamp of the most recent authenticated request made with this key. `null` if the key has never been used.
        </ResponseField>

        <ResponseField name="expires_at" type="string">
          ISO 8601 expiry timestamp. `null` if the key has no expiry.
        </ResponseField>

        <ResponseField name="is_active" type="boolean" required>
          `true` if the key can authenticate requests. `false` if the key has been revoked or has expired.
        </ResponseField>

        <ResponseField name="is_expired" type="boolean" required>
          `true` if the key's `expires_at` is in the past.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<Tip>
  During a key rotation, poll `GET /v2/api-keys` and watch `last_used_at` on the new key. Once it starts updating, you know live traffic has moved to the new key and it is safe to revoke the old one.
</Tip>

## Example

<CodeGroup>

```bash curl
curl "$API_BASE/v2/api-keys" \
  -H "Authorization: Bearer $USER_ACCESS_TOKEN"
```

```javascript JavaScript
const response = await fetch(`${API_BASE}/v2/api-keys`, {
  headers: {
    Authorization: `Bearer ${USER_ACCESS_TOKEN}`,
  },
});

const { payload } = await response.json();
console.log(payload.keys);
```

</CodeGroup>
